# RishavendraOS Content Architecture Specification v1.0

## Purpose

This document defines the information architecture of RishavendraOS.

Most portfolio websites organize content around projects.

RishavendraOS organizes content around intelligence.

The purpose is not to showcase what was built.

The purpose is to reveal:

- How ideas emerge
- How decisions are made
- How systems are designed
- How failures are handled
- How learning compounds

The visitor should leave understanding:

> How Rishavendra thinks.

rather than simply:

> What Rishavendra built.

---

The entire experience revolves around one central story:

# Road Detection World

## Purpose

Demonstrate practical AI experimentation.

---

# Project Name

Drone-Based Binary Terrain Mapping with Road Measurement

---

# Project Summary

A computer vision system that classifies aerial imagery into road and non-road regions while estimating real-world infrastructure measurements.

---

# Problem

Road infrastructure measurements are traditionally time-consuming.

The project explores whether drone imagery and AI can assist in estimating infrastructure characteristics.

---

# Dataset Reality

Only 86 aerial images were available.

This created major limitations.

---

# Important Accuracy Statement

Road measurements produced reasonably accurate estimations.

Terrain reconstruction could not be fully achieved because the dataset was insufficient for reliable mapping.

---

# Final Content Rule

Every piece of content must answer one of four questions:

Why does it matter?

How does it work?

What was learned?

If a section does not answer one of these questions, it should not exist.

---

# Success Criteria

The content architecture succeeds when visitors leave with three conclusions:

1. This person builds real systems.

2. This person understands how those systems work.

3. This person continuously learns and improves.

The visitor should remember the thinking process behind the projects, not merely the project titles.
